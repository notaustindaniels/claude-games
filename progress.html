Debug findings worth recording: (1) three r181 colorNode output clamps negatives — all GPGPU passes must use fragmentNode; (2) WebGL-backend TSL compute reads element(j) as the invocation's OWN element (transform-feedback attribute binding) — silent wrong gathers, so butterflies live in fragment passes; (3) readbacks return rows top-down; (4) HalfFloat readbacks return raw Uint16 bits; (5) the post pass's screenUV has v=0 at top — my ray reconstruction was vertically inverted, misclassifying above/below water per pixel (the "black wedge" and "impossible sand" artifacts in M6); (6) chop-folded wave triangles can sweep across the camera — horizontal chop is now tapered to zero within ~4–24 m of the eye; (7) the surface's above/below branch now keys on camera submersion (fragment-side heightAt test), not frontFacing, so folded faces can't flip shading mid-air.
